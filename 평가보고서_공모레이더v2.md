## 🪞 자아비판 (Claude의 솔직한 자기평가)

### 내가 잘못 처리한 것들

**1. NTIS/K-Startup 아키텍처 분리 실수 (가장 심각)**
GUI(`CollectionWorker` 내 인라인)와 Scheduler(`scheduler.py`) 사이의 수집 로직이 분리됨. 결과적으로 자동화 스케줄러로 실행할 때 NTIS와 K-Startup 공고가 완전히 누락된다. 이는 처음 설계 단계에서 "모든 수집기는 독립 파일"이라는 원칙을 끝까지 지켜야 했는데, 빠른 구현을 위해 인라인으로 처리한 결과다.

**2. NTIS URL 실제 검증 없이 제출**
`https://www.ntis.go.kr/rndgate/eg/is/pgBizInfo/listBizInfo.do` — 실측 HTTP 404. 코드 작성 시 URL을 실제로 테스트하지 않고 문서 기반으로 추정해 작성했다. 배포 전 필수 확인 단계였다.

**3. "정부조직 기준 전체 부처/처/청/위원회" 요청 불완전 이행**
사용자가 명확하게 요청했음에도 46개 기관 중 15개(33%)만 포함했다. 나머지 31개는 "대학과 관련성 낮다"는 자의적 판단으로 누락했는데, 이는 사용자의 요청 범위를 초과한 판단이었다.

**4. config.yaml 자격증명 보안 설계 방기**
`.env` 파일이 requirements.txt에 포함되어 있음에도 실제로 활용하지 않고 config.yaml에 평문 입력을 유도했다. 보안 설계를 마지막 단계로 미루다가 결국 미처리됨.

**5. test_all.py 테스트 범위 불충분**
수집기 7개에 대한 단위 테스트 없음. 네트워크 연결 확인(ping 수준)과 DB/Matcher 테스트만 존재. 각 수집기의 HTML 파싱 로직이 올바른지 검증하는 fixture 기반 테스트가 없음.

**6. Write 도구 한계를 미리 인지하고 모든 파일을 heredoc으로 작성했어야 했음**
첫 scheduler.py 작성 시 Write 도구 한국어 f-string 잘림 버그로 2번 재작업이 필요했다. 처음부터 bash heredoc을 사용했다면 불필요한 검증 사이클을 줄일 수 있었다.

### 잘한 것들 (솔직하게)
- `safe_collect()` 패턴 / `sys.frozen` / `daemon=True` 조합은 실무 수준.
- 네이비+골드 브랜딩 Excel 3시트 구조는 실제 보고 용도에 적합.

### Gemini 1.5 Pro의 예상 지적

> *"Rate limiting이 전혀 없습니다. 21개 정부 사이트에 1초 이내 연속 요청 시 IP 차단 가능성이 높습니다. 또한 robots.txt 미확인 크롤링은 공공기관 대상이라도 법적 리스크가 있습니다. `scrapy`의 `DOWNLOAD_DELAY` 설정이나 최소 `time.sleep(0.5)` 추가를 권장합니다. 아울러 BeautifulSoup CSS 선택자가 사이트 리뉴얼에 취약하므로 공공데이터포털 API를 우선적으로 활용하고 크롤링은 fallback으로만 사용하는 구조를 권장합니다."*

### Copilot (Microsoft)의 예상 지적

> *"Azure를 활용하지 않는 선택은 이해하지만, Windows 환경에서 `subprocess.Popen(['ollama', 'serve'])` 호출 시 `creationflags=0x00000008 | 0x08000000` 플래그가 올바릅니다. 다만 `build_exe.bat`에서 `--name "공모레이더"` 한글 파일명은 일부 Windows 빌드 환경에서 인코딩 오류를 발생시킵니다. 추가로, `tkinter`의 `ttk.Combobox`와 `ttk.Treeview`에서 Windows 11 기본 테마와 충돌이 발생하면 `root.tk.call('source', 'azure.tcl')` 같은 커스텀 테마 적용을 고려하세요."*
